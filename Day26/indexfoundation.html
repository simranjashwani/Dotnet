Introduction to HTTP

HTTP (HyperText Transfer Protocol) is the communication protocol that powers the web. When your browser loads a page, it sends an HTTP request to a server, and the server sends back an HTTP response containing HTML.

Request: Browser asks — GET /index.html HTTP/1.1

Response: Server replies — 200 OK + the HTML document

Think of it like: You walk into a library (browser), ask for a book (request), and the librarian hands it to you (response).

HTTP Methods and Status Codes

HTTP Methods describe the action the browser wants to perform. Status Codes tell you what happened on the server side — success, failure, or something in between.

Method Purpose Real World Analogy
GET Fetch a resource Reading a menu
POST Send new data Placing an order
PUT Replace existing data Replacing an order
PATCH Update part of data Modifying one item in order
DELETE Remove a resource Cancelling an order
Status Code Meaning
2XX OK — everything worked
301 Moved Permanently — page has a new URL
404 Not Found — page doesn't exist
500 Server Error — something broke on the server

HTML Document Structure and DOM

Every HTML file follows a standard structure — a doctype declaration, a root html element, a head for metadata, and a body for visible content. The browser reads this and builds the DOM — a live tree of all elements that JavaScript can interact with. 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello World</h1>
    <p>This is my first webpage.</p>
  </body>
</html>

DOM Tree: html → head + body → h1 + p

Think of the DOM like a family tree. Every element has a parent, and can have children and siblings.

HTML Elements and Attributes

An HTML element is an opening tag, its content, and a closing tag. Attributes are extra information placed inside the opening tag that modify or describe the element.

Basic element: This text is bold

Element with attribute: Visit MDN

Image with attributes: 

<a href="https://developer.mozilla.org" target="_blank">Visit MDN</a>
<img src="photo.jpg" alt="Description of image" />

Inline and Block Elements

Block elements take up the full width of their container and always start on a new line. Inline elements only take up as much space as their content and sit within the flow of text.

Block elements — each on its own line:

I am a DIV — block element
I am another DIV — I pushed to a new line

Inline elements — sitting side by side:

This is text with a span and bold text and italic text all sitting inline.

Semantic HTML5 Elements

Semantic elements clearly describe their meaning to both the browser and the developer. Instead of using generic divs everywhere, semantic tags tell you what role that section plays on the page.

<header> — site logo, nav, top of page
<nav> — navigation links
<article> — self-contained content like a blog post
<aside> — sidebar content
<section> — a thematic grouping of content
<footer> — bottom of page, copyright, links

HTML Forms

A form is a container that collects user input and submits it to a server. The action attribute tells the form where to send the data, and the method tells it how — GET or POST. 

<form action="/submit" method="POST">
  <label for="name">Your Name</label>
  <input type="text" id="name" name="name" />
  <button type="submit">Submit</button>
</form>

HTML5 Validation

HTML5 has built-in form validation that runs in the browser before any data is sent to the server. Using attributes like required, minlength, and pattern, you get validation for free — no JavaScript needed.

Try clicking Submit without filling fields — browser validation kicks in automatically.
