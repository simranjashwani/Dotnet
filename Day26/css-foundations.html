Overview of CSS

CSS (Cascading Style Sheets) is the language that controls how HTML elements look. It separates presentation from structure — HTML says what something is, CSS says how it looks.

This paragraph is styled with CSS — color, font size, and background all come from the stylesheet.

Inline, Internal, and External Stylesheets

CSS can be written in three places — directly on an element (inline), inside a style tag in the HTML head (internal), or in a separate .css file linked to the HTML (external). External is the professional standard.

Inline style — written directly on this element using the style attribute.

Internal style — would live inside a <style> tag in the <head>.

CSS Selectors (Element, Class, ID)

Selectors tell CSS which HTML elements to target. An element selector targets all tags of that type, a class selector targets elements sharing a class, and an ID selector targets one unique element.

Element selector — targets all <p> tags

Class selector — targets elements with class="highlight-class"

ID selector — targets only this element with id="unique-id"

CSS Box Model

Every HTML element is a rectangular box. The box model describes four layers around the content — padding (inner space), border (the edge), and margin (outer space). Understanding this is fundamental to layout.

CSS Properties and Values

A CSS rule is made of a property (what to change) and a value (what to change it to). Properties cover everything from color and font to spacing, size, and display behavior.

color: steelblue — changes text color

background-color: #fff3cd — changes background

font-size: 1.3rem — changes text size

font-weight: bold — makes text bold

letter-spacing: 0.2em — spreads letters apart

text-transform: uppercase — forces caps

border-radius: 8px — rounds corners

Cascading and Specificity

When multiple CSS rules target the same element, the cascade determines which one wins. Specificity is the scoring system — IDs score highest, then classes, then element tags. When scores tie, the last rule wins.

Styled by element selector (lowest specificity)

Class overrides element selector (higher specificity)

ID overrides class (highest specificity)

Media Queries

Media queries apply CSS rules only when certain conditions are met — like screen width, orientation, or print mode. They are the mechanism behind responsive design.

Resize your browser to see this change.

@media (max-width: 768px) {
  .container { flex-direction: column; }
}
@media (min-width: 1200px) {
  .container { max-width: 1140px; }
}
@media print {
  nav, footer { display: none; }
}

Mobile-First Approach

Mobile-first means you write your base CSS for small screens, then use min-width media queries to progressively enhance the layout for larger screens. It ensures the core experience works on the smallest device first.

/* Base styles — mobile */
.card {
  width: 100%;
  padding: 1rem;
}

/* Tablet and above */
@media (min-width: 600px) {
  .card { width: 48%; }
}

/* Desktop and above */
@media (min-width: 1024px) {
  .card { width: 30%; }
}

Mobile-first vs desktop-first: Start small, scale up. Not the other way around.

Viewport Meta Tag

The viewport meta tag tells mobile browsers not to zoom out and fake a desktop layout. Without it, your perfectly written responsive CSS will be ignored on phones. It belongs in every HTML file.

<!-- Always include this in your <head> -->
<meta name="viewport" content="width=device-width, initial-scale=1.0" />

<!-- width=device-width: match screen width -->
<!-- initial-scale=1.0: don't zoom in or out on load -->

Without this tag, a phone renders the page as if it were 980px wide and then scales it down — making text tiny and media queries useless.

Responsive Units (rem, em, %, vw, vh)

Responsive units scale relative to something else — the root font size, the parent element, or the viewport. Using them instead of fixed pixels makes layouts naturally fluid and accessible.

font-size: 16px — fixed, never changes

font-size: 1.5rem — 1.5x the root font size (usually 24px)

font-size: 1.2em — 1.2x the parent's font size

width: 60% — 60% of the parent container
width: 50vw — 50% of the viewport width
height: 10vh — 10% of the viewport height
Unit Relative To Best For
px Absolute Borders, fine detail
rem Root font size Font sizes, spacing
em Parent font size Component-scoped sizing
% Parent dimension Fluid widths
vw Viewport width Full-width sections
vh Viewport height Full-height hero sections
